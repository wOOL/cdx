# 12. Explanation of symbols

Symbols used throughout the application:

| Symbol | Meaning |
|--------|---------|
| ⚠ (yellow) | Advisory warning — data quality, design rules, verification reminders. Read the tooltip/banner text. |
| ⚠ (red text/chips) | Active safety-distance or collision violation; details in the status bar and report. |
| ✓ (green, stage bar / EASY rail) | Stage completed for the active plan. |
| 🔒 / *locked* badge | Write-protected object: locked plan or locked dataset. Unlocking is an explicit, audited action. |
| *sent* badge | Plan was transferred to a contact and is write-protected; duplicate it to continue planning. |
| *approved* | Plan is finalized; guide export unlocked. Revoking approval re-blocks export. |
| *Guide outdated* | Planning changed after guide generation; regenerate before export. |
| ⏳ / ✓ / ⚠ (dataset card) | Dataset processing status: running / ready / failed. |
| 👁 | Toggle object visibility in the views. |
| 🗑 | Delete object (confirmation required where destructive). |
| ⭐ | Favorite implant line in the library picker. |
| 🌐 | Regional availability of a catalog entry. |
| ⚙ (library card) | Technical information of an implant line (hover); links to documentation where available. |
| 😎 | Patient record is anonymized. |
| ⌖ | Align cross-section views to the selected implant's axis. |
| 📷 | Save view snapshot to the image library (Alt-click: download). |
| ⤢ / Esc | Maximize a view / restore the grid. |
| ① ② ③ … | Numbered callouts in this manual's annotated screenshots. |

Keyboard shortcuts referenced in this manual: **F1** help · **F8** screen copy ·
**Ctrl +/− / Ctrl 0** zoom / reset · **Ctrl Z / Ctrl Shift Z** undo / redo ·
**Esc** restore maximized view / close dialogs.
